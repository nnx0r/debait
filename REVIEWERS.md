# Build instructions for AMO reviewers

This document is for the Firefox Add-ons (AMO) review team. It explains how to
reproduce the submitted extension package from source. The end-user/developer
docs are in [README.md](README.md).

The submitted add-on contains **one bundled script** (`content-scripts/content.js`),
produced from TypeScript source by [WXT](https://wxt.dev) (which uses Vite to
transpile and minify). Everything else in the package is static or generated
text (manifest, `rules.json`, `_locales/`, icons) and is human-readable as-is.

## Build environment

The submitted package was built with:

| Tool             | Version used | Minimum required        |
| ---------------- | ------------ | ----------------------- |
| Node.js          | 24.17.0      | 20.12.0 (WXT engine req)|
| npm              | 11.13.0      | bundled with Node       |
| Operating system | macOS        | any (Linux/Windows OK)  |

- Node version is also pinned in [`.nvmrc`](.nvmrc) — `nvm use` will select it.
- The package manager is **npm** (lockfile: `package-lock.json`). Do not use
  pnpm or yarn — the lockfile is for npm.
- **No environment variables, `.env` file, API keys, or network access** are
  required to build. The build is fully offline after `npm ci`.

## Steps to reproduce

From the root of this source archive:

```bash
nvm use            # optional; selects Node 24.17.0 from .nvmrc
npm ci             # deterministic install from package-lock.json
npm run build:firefox
```

The unpacked, reviewable extension is written to:

```
.output/firefox-mv3/
```

This directory is byte-for-byte the contents of the submitted add-on. To also
produce the exact submitted `.zip` (and a fresh copy of this source archive):

```bash
npm run zip:firefox
# -> .output/debait-<version>-firefox.zip   (the add-on package)
# -> .output/debait-<version>-sources.zip   (this source archive)
```

## Source → bundle mapping

| Submitted file                  | Source                                            |
| ------------------------------- | ------------------------------------------------- |
| `manifest.json`                 | generated by WXT from [wxt.config.ts](wxt.config.ts) |
| `content-scripts/content.js`    | [entrypoints/content.ts](entrypoints/content.ts) (transpiled + minified by Vite) |
| `rules.json`                    | [public/rules.json](public/rules.json) (copied verbatim) |
| `_locales/<locale>/messages.json` | [public/_locales/](public/_locales/) — generated from [scripts/gen-i18n.mjs](scripts/gen-i18n.mjs) via `node scripts/gen-i18n.mjs` |
| `icon/*.png`                    | [public/icon/](public/icon/) (copied verbatim)    |

`content.js` is the only minified artifact; its complete unminified source is
`entrypoints/content.ts`. No code is fetched at runtime and no remote code is
executed.

## What the extension does (for review context)

DeBait redirects YouTube thumbnail image requests to the video's real
auto-generated first frame, using a **static `declarativeNetRequest` ruleset**
([public/rules.json](public/rules.json)). The content script
([entrypoints/content.ts](entrypoints/content.ts)) is only a fallback that
detects when a redirected frame 404s and restores the original thumbnail.

- Host access is limited to `*.youtube.com` and `*.ytimg.com`.
- Static DNR rulesets require **Firefox 128+**, declared via
  `browser_specific_settings.gecko.strict_min_version` in the manifest.
- The extension collects no data (`data_collection_permissions: { required: ['none'] }`).
